# Medication Assistant

A Flask + Supabase web app that combines:
- Medication-focused AI chat (Gemini / OpenRouter)
- Calendar-based medicine reminders with notifications
- Chat session history and PDF export

## Features

- Medication-focused chatbot with two providers:
  - Gemini
  - OpenRouter
- Prompt-restricted assistant behavior for medical context
- Session management:
  - Create, rename, star, delete session
  - Clear all chat history
- Attachment support:
  - PDF and text extraction
  - Image attachments in chat
- Reminder system:
  - Sidebar quick calendar
  - Dedicated calendar view
  - Date/time reminders with repeat modes
  - In-app and browser notifications
  - Mark-as-taken tracking
- Smart title generation from first user message
- Export chat as PDF
- Mobile-friendly UI

## Tech Stack

- Backend: Flask
- Database: Supabase (PostgreSQL)
- Frontend: Vanilla JavaScript, HTML, CSS
- PDF: reportlab, html2pdf, pdf.js

## Project Structure

- server.py: Flask app and API routes
- templates/index.html: Main UI
- static/js/main.js: Client logic (chat, calendar, reminders)
- static/js/gemini.js: Gemini integration
- static/js/openrouter.js: OpenRouter integration
- static/css/styles.css: UI styles
- api/index.py: Vercel entrypoint
- requirements.txt: Python dependencies

## Prerequisites

- Python 3.10+
- Conda environment named ai (recommended in this project)
- Supabase project
- Gemini and/or OpenRouter API key

## Environment Variables

Create a .env file in the project root:

GEMINI_API_KEY=your-gemini-api-key
OPENROUTER_API_KEY=your-openrouter-api-key
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your-supabase-anon-key
PORT=8000

Important:
- Do not store API keys in frontend files.
- Do not put API keys in browser localStorage.
- This project now uses server-side AI proxy routes so secrets stay private.

## Supabase Setup

Use the SQL below in Supabase SQL Editor.

CREATE TABLE chats (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    title TEXT DEFAULT 'New Chat',
    is_favorite BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE messages (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    chat_id BIGINT NOT NULL REFERENCES chats(id) ON DELETE CASCADE,
    role TEXT NOT NULL,
    content TEXT DEFAULT '',
    image_data TEXT,
    provider_mode TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_messages_chat_id ON messages(chat_id);

## API Overview

Sessions
- GET /api/sessions
- POST /api/sessions
- PATCH /api/sessions/<sid>
- DELETE /api/sessions/<sid>
- DELETE /api/sessions (clear all sessions)
- POST /api/sessions/<sid>/generate-title

Messages
- GET /api/sessions/<sid>/messages
- POST /api/sessions/<sid>/messages

Export
- GET /api/sessions/<sid>/export

AI Proxy (server-side keys)
- POST /api/ai/gemini
- POST /api/ai/openrouter

## Secure Migration Checklist

If your keys were exposed earlier, do this once:

1. Rotate Gemini and OpenRouter keys in their dashboards.
2. Remove old keys from local files and browser storage.
3. Set new keys only in `.env` (local) and Vercel Environment Variables (production).
4. Restart local server and redeploy Vercel.

## Deploy (Vercel)

1. Push code to GitHub.
2. Import the repo in Vercel.
3. In Vercel Project Settings -> Environment Variables, add:
  - GEMINI_API_KEY
  - OPENROUTER_API_KEY
  - SUPABASE_URL
  - SUPABASE_ANON_KEY
  - PORT (optional)
4. Keep `vercel.json` as-is (entrypoint: `api/index.py`).
5. Deploy.

## Supabase Setup (Production)

1. Create project in Supabase.
2. Run the SQL from this README in SQL Editor to create `chats` and `messages`.
3. Copy values from Project Settings -> API:
  - Project URL -> `SUPABASE_URL`
  - anon public key -> `SUPABASE_ANON_KEY`
4. Add these values to local `.env` and Vercel Environment Variables.
5. Verify by creating a chat in the app and checking rows in `chats` and `messages`.

## Notes

- Browser notifications require permission.
- Reminder auto-extraction from text is heuristic and may require manual edits.
- Keep .env private and never commit real keys.
