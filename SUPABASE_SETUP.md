## 📋 Database Schema Reference

Your Supabase database needs these two tables:

### Table: `chats`
```sql
CREATE TABLE chats (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    title TEXT DEFAULT 'New Chat',
    is_favorite BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### Table: `messages`
```sql
CREATE TABLE messages (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    chat_id BIGINT REFERENCES chats(id) ON DELETE CASCADE,
    role TEXT NOT NULL,
    content TEXT DEFAULT '',
    image_data TEXT,
    provider_mode TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

**Note:** Cascade deletion (ON DELETE CASCADE) is set so deleting a chat automatically deletes all its messages.

## 🚀 Step-by-Step Supabase Setup

### Step 1: Create a Supabase Account
1. Go to [https://supabase.com](https://supabase.com)
2. Click "Sign Up" (use GitHub or email)
3. Verify your email

### Step 2: Create a New Project
1. In Supabase dashboard, click "New Project"
2. Choose a project name: `medication-chatbot` (or similar)
3. Set a strong database password (save it somewhere safe)
4. Select your region (closest to your location for better performance)
5. Click "Create new project" and wait (~2 minutes)

### Step 3: Create Database Tables

#### Option A: Using Supabase Dashboard (Easiest)
1. Go to your project dashboard
2. Click "SQL Editor" in the left sidebar
3. Click "New Query"
4. Copy and paste this SQL:

```sql
-- Create chats table
CREATE TABLE chats (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    title TEXT DEFAULT 'New Chat',
    is_favorite BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Create messages table
CREATE TABLE messages (
    id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
    chat_id BIGINT NOT NULL REFERENCES chats(id) ON DELETE CASCADE,
    role TEXT NOT NULL,
    content TEXT DEFAULT '',
    image_data TEXT,
    provider_mode TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Create index on chat_id for faster queries
CREATE INDEX idx_messages_chat_id ON messages(chat_id);
```

5. Click "Run" (or press Ctrl+Enter)
6. Should see: "Success. No rows returned"

#### Option B: Using Table Editor
1. Go to "Table Editor" in left sidebar
2. Click "Create a new table"
3. Fill in details for `chats` table:
   - Name: `chats`
   - Columns:
     - `id` (BigInt, Primary Key, Enable Identity)
     - `title` (Text, Default: 'New Chat')
     - `is_favorite` (Boolean, Default: false)
     - `created_at` (Timestamp, Default: now())

4. Repeat for `messages` table:
   - Name: `messages`
   - Columns:
     - `id` (BigInt, Primary Key, Enable Identity)
     - `chat_id` (BigInt, Foreign Key → chats.id, Cascade delete)
     - `role` (Text, Required)
     - `content` (Text, Default: '')
     - `image_data` (Text, Allow NULL)
     - `provider_mode` (Text, Allow NULL)
     - `created_at` (Timestamp, Default: now())

### Step 4: Get Supabase Credentials

1. In Supabase dashboard, click "Project Settings" (bottom left)
2. Click "API" tab
3. Copy these values:
   - **Project URL** → SUPABASE_URL
   - **Anon public key** → SUPABASE_ANON_KEY

Example:
```
SUPABASE_URL: https://xxxxxxxxxxxx.supabase.co
SUPABASE_ANON_KEY: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Step 5: Update Your .env File

Create or edit `.env` file in your project root:

```env
# Medication Chatbot Configuration
GEMINI_API_KEY=your-gemini-key-here
OPENROUTER_API_KEY=your-openrouter-key-here

# Supabase Configuration
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your-anon-key-here

PORT=8000
```

⚠️ **IMPORTANT:** Never commit `.env` to Git. It's already in `.gitignore`.

### Step 6: Install Dependencies

```bash
conda activate ai
pip install -r requirements.txt
```

This installs:
- Flask 3.1.3
- reportlab 4.2.2
- python-dotenv 1.0.1
- **supabase 2.5.0** (new)

### Step 7: Test the Connection

Run your server:
```bash
conda activate ai
python server.py
```

You should see:
```
  Medication Assistant running at http://localhost:8000
```

✅ If no errors, Supabase is connected!

## 🔒 Row-Level Security (Optional but Recommended)

For production, enable Row-Level Security (RLS) to protect user data:

1. In Supabase, go to "Authentication" → "Policies"
2. Click on `chats` table
3. Click "New Policy"
4. Select "Enable RLS"
5. Create policy to restrict users to their own chats

For now, this is optional since the chatbot doesn't have user authentication yet.

## 🧪 Code Logic Verification

### Database Operations Mapping

| Operation | Old (SQLite) | New (Supabase) | Status |
|-----------|------------|----------------|--------|
| Create chat | `conn.execute("INSERT...")` | `supabase.table('chats').insert()` | ✅ |
| Read chats | `conn.execute("SELECT...")` | `supabase.table('chats').select()` | ✅ |
| Update chat | `conn.execute("UPDATE...")` | `supabase.table('chats').update().eq()` | ✅ |
| Delete chat | `conn.execute("DELETE...")` | `supabase.table('chats').delete().eq()` | ✅ |
| Save message | `conn.execute("INSERT...")` | `supabase.table('messages').insert()` | ✅ |
| Get messages | `conn.execute("SELECT...")` | `supabase.table('messages').select()` | ✅ |
| Export PDF | `conn.execute("SELECT...")` | `supabase.table('chats/messages').select()` | ✅ |

### Error Handling

All routes now include try-catch blocks:
```python
try:
    response = supabase.table('chats').select('*').execute()
    return jsonify(response.data)
except Exception as e:
    return jsonify({'error': str(e)}), 500
```

### API Response Format

Supabase returns data as a list of dictionaries:
```python
response.data = [
    {'id': 1, 'title': 'Medication Info', 'is_favorite': False, 'created_at': '2024-01-01T12:00:00'},
    {'id': 2, 'title': 'Drug Interactions', 'is_favorite': True, 'created_at': '2024-01-02T12:00:00'}
]
```

This directly converts to JSON for API responses. ✅

## 🐛 Troubleshooting

### Error: "SUPABASE_URL and SUPABASE_ANON_KEY must be set"
**Solution:** Make sure `.env` file is in the project root with correct values.

### Error: "Network error connecting to Supabase"
**Possible causes:**
- Invalid SUPABASE_URL (typo)
- Firewall blocking request
- Supabase server down

**Solution:**
- Double-check credentials in `.env`
- Test with curl: `curl https://your-project.supabase.co/auth/v1/health`

### Error: "Table 'chats' does not exist"
**Solution:**
- Go to Supabase SQL Editor
- Run the table creation SQL again
- Verify tables exist in Table Editor

### Error: "Permission denied" when inserting
**Solution:**
- This usually means RLS is enabled. Either:
  - Disable RLS for now: Go to Table Editor → click table → Policies → Toggle off
  - Or create proper RLS policies
